# Chrome Start Page Plugin - 安装测试指南

## 🧪 快速测试指南

### 1. 准备环境
确保你有以下环境：
- Chrome浏览器 (版本88+)
- 开发者模式已启用

### 2. 安装扩展
```bash
# 1. 进入chrome-extension目录
cd chrome-extension

# 2. 在Chrome中打开扩展管理页面
# 输入: chrome://extensions/

# 3. 开启"开发者模式" (右上角开关)

# 4. 点击"加载已解压的扩展程序"
# 选择chrome-extension文件夹
```

### 3. 验证安装
检查以下项目：
- [ ] 新标签页显示自定义首页 (不是Chrome默认页面)
- [ ] 浏览器工具栏显示扩展图标
- [ ] 页面顶部有"我的首页"标题
- [ ] 有"添加分组"和"添加书签"按钮

### 4. 功能测试

#### 4.1 测试分组管理
- [ ] 点击"添加分组"按钮

- [ ] 修改分组名称为"已编辑分组"
- [ ] 确认名称更新成功

#### 4.2 测试书签操作

- [ ] 确认书签出现在分组中
- [ ] 点击书签标题，确认能正常打开网页
- [ ] 点击书签的删除按钮
- [ ] 确认书签被删除

#### 4.3 测试数据持久化

## 🔧 常见问题解决

### 问题1: 扩展无法加载
**症状**: 点击"加载已解压的扩展程序"后显示错误
**解决**: 
1. 检查manifest.json文件是否存在且格式正确
2. 确保文件路径不包含中文字符
3. 重新下载完整的项目文件

### 问题2: 新标签页没有变化
**症状**: 仍然显示Chrome默认新标签页
**解决**:
1. 检查扩展是否已启用
2. 在扩展管理页面点击"刷新"按钮
3. 重启浏览器

### 问题3: 样式显示异常
**症状**: 页面样式错乱或空白
**解决**:
1. 检查CSS文件路径是否正确
2. 确认reset.css和styles.css都已加载
3. 清理浏览器缓存

### 问题4: 数据无法保存
**症状**: 刷新页面后数据丢失
**解决**:
1. 检查Chrome Storage API权限
2. 查看控制台错误信息
3. 确保manifest.json包含storage权限

### 问题5: 书签图标不显示
**症状**: 书签旁边没有图标或显示默认图标
**原因**: 这是正常现象，图标获取功能在P2版本中实现
